# active-memory

Demonstrates Active Memory (Phase 7) — a blocking pre-send recall that
prepends relevant facts as an `<active-memory>` block at the top of the
LLM system prompt.

## Run

```bash
pnpm install --ignore-workspace
cp .env.example .env
pnpm dev
```

## What it does

1. Writes 3 seed facts to `.theokit/memory/MEMORY.md`.
2. Creates an agent with `memory.activeRecall.enabled: true` AND
   `memory.index.tools: false` (tools disabled — only the recall block).
3. Sends `"What's the magic-number?"`. Before the LLM call, the SDK
   queries the index with the user message and injects the top hits as
   a `<active-memory>` block above context/skills/memory.
4. The model answers using only the injected block.

## Configuration

```ts
memory: {
  enabled: true,
  activeRecall: {
    enabled: true,
    queryMode: "message",   // or "recent" / "full"
    maxSummaryChars: 400,
    timeoutMs: 10000,       // 15s default
    persistTranscripts: false,
  },
}
```

## Resilience

- **Timeout** — if the index search exceeds `timeoutMs`, the recall
  returns `status: "timeout"` and the send proceeds without the block.
- **Circuit breaker** — 3 consecutive timeouts for the same agent trip
  a 60-second cooldown; recall skipped during cooldown.
- **Cache** — 15s TTL keyed by `sha256(userText + queryMode)`. Two
  identical sends within the TTL share one search.

## v1 limitation

The block is generated by a deterministic search (top 5 hits formatted
as a markdown list). The LLM-mediated `mode: "subagent"` flavor (a tiny
dedicated model curates the summary) is Phase 7.1 future work.
